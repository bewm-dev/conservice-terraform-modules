# Conservice Terraform Modules

Shared Terraform modules for the Conservice AWS greenfield platform.

## Modules

### Platform Modules (SRE-owned)

Used by `conservice-aws-platform` account configs. These cover patterns that community modules don't handle.

| Module | Purpose | Used By |
|--------|---------|---------|
| `conservice-account-iam-baseline` | Per-account baseline IAM (TF execution, ECR pull, Aurora access roles) | All account `global/iam` |
| `conservice-eks-addons` | Pod Identity roles for LBC, ExternalDNS, ESO, Karpenter, Container Insights | All EKS cluster configs |
| `conservice-argocd` | ArgoCD bootstrap via Helm (minimal install + Dex SSO secrets) | Platform eks-mgmt |
| `conservice-argocd-add-cluster` | Register remote EKS cluster with ArgoCD management cluster | Production (Phase 7.3) |
| `conservice-vpn-peer` | Site-to-site VPN peer on Transit Gateway with ephemeral PSKs | Platform network |

### App-Level Modules (dev team-facing)

| Module | Purpose |
|--------|---------|
| `conservice-app-resources` | Orchestrator — reads YAML config, provisions S3 (community module), SQS, SNS, databases |
| `conservice-app-database` | PostgreSQL database + IAM-auth roles + grants inside shared Aurora cluster |

`conservice-app-resources` is the main entry point for app teams. It calls `conservice-app-database` internally for database provisioning, and uses `terraform-aws-modules/s3-bucket/aws` for S3 buckets. SQS and SNS use native resources (simple enough to not need a wrapper).

Dev teams edit YAML in their app repo's `infra/` directory. The scaffolder generates a thin `main.tf` that calls `conservice-app-resources` — devs never touch HCL.

### Community Modules (used directly in account configs)

Complex infrastructure components use [terraform-aws-modules](https://github.com/terraform-aws-modules) from the Terraform Registry. See `APPLY-ORDER.md` in `claude-code-conservice` for the full list.

| Module | Used For |
|--------|----------|
| `terraform-aws-modules/vpc/aws` | All VPCs + VPC endpoints |
| `terraform-aws-modules/transit-gateway/aws` | Platform TGW hub |
| `terraform-aws-modules/eks/aws` | All EKS clusters + Karpenter submodule |
| `terraform-aws-modules/rds-aurora/aws` | All Aurora clusters |
| `terraform-aws-modules/acm/aws` | Platform ACM certs |
| `terraform-aws-modules/route53/aws` | Platform Route53 zones |
| `terraform-aws-modules/s3-bucket/aws` | Log archive + app-level S3 buckets |
| `terraform-aws-modules/iam/aws` | GitHub OIDC provider + role |

## App-Level Config Pattern

Dev teams write YAML, not Terraform. The 3-layer config merge:

```text
my-app/
├── src/
├── deploy/              # K8s manifests (ArgoCD)
└── infra/
    ├── main.tf          # Generated by scaffolder — calls conservice-app-resources
    ├── provider.tf      # Generated by scaffolder — backend + provider config
    ├── base.yaml        # Team, domain, portfolio (required)
    ├── database.yaml    # Database config (optional)
    ├── s3.yaml          # S3 bucket config (optional)
    ├── sqs.yaml         # SQS queue config (optional)
    ├── sns.yaml         # SNS topic config (optional)
    └── envs/
        ├── dev.yaml     # Dev overrides (optional)
        ├── stg.yaml     # Staging overrides (optional)
        └── prod.yaml    # Prod overrides (optional)
```

### Resource naming

| Resource | Pattern | Example |
|----------|---------|---------|
| S3 buckets | `conservice-{env}-{app}-{key}` | `conservice-prod-rates-exports` |
| SQS queues | `csvc-{env}-{region}-{app}-{key}-queue` | `csvc-prod-use1-rates-events-queue` |
| SQS DLQs | `csvc-{env}-{region}-{app}-{key}-dlq` | `csvc-prod-use1-rates-events-dlq` |
| SNS topics | `csvc-{env}-{region}-{app}-{key}-topic` | `csvc-prod-use1-rates-alerts-topic` |
| Databases | YAML key | `rates` |

## Usage

### Platform module (in conservice-aws-platform)

```hcl
module "account_base" {
  source = "git::https://github.com/shawnpetersen/conservice-terraform-modules.git//modules/conservice-account-iam-baseline?ref=conservice-account-iam-baseline/v1.0.0"

  env              = "dev"
  aws_account_id   = "106751264134"
  tools_account_id = "626209130023"
}
```

### App-level infra (in app repo)

```hcl
# infra/main.tf — generated by scaffolder, devs don't edit
module "resources" {
  source = "git::https://github.com/shawnpetersen/conservice-terraform-modules.git//modules/conservice-app-resources?ref=conservice-app-resources/v1.0.0"

  app_name    = "rates"
  env         = var.env
  region      = var.region
  config_path = path.module
}
```

```yaml
# infra/base.yaml — dev team edits this
name: rates
team: billing
domain: billing
portfolio: billing
```

```yaml
# infra/s3.yaml — dev team edits this
buckets:
  exports:
    versioning: true
  attachments:
    versioning: false
```

## Versioning

Use directory-scoped git tags: `?ref=conservice-app-resources/v1.0.0`. Never reference `?ref=main` in production.
